## Instructions

1. **Page Layout**
   - Structure pages with semantic HTML
   - Use responsive grids and flex layouts
   - Organize content hierarchically

2. **Components**
   - Create reusable UI components
   - Use props/state for dynamic behavior
   - Follow component composition best practices

3. **Styling**
   - Apply CSS, Tailwind, or styled-components
   - Use consistent spacing, typography, and colors
   - Implement dark/light mode if needed

4. **Interactivity**
   - Handle events (click, hover, input)
   - Manage local state or global state
   - Use animations for smooth UI feedback

5. **Integration**
   - Connect frontend with backend APIs
   - Handle loading and error states
   - Ensure accessibility and SEO best practices

## Best Practices
- Mobile-first design approach  
- Keep components small and reusable  
- Follow consistent naming conventions  
- Optimize images and assets  
- Use semantic HTML for accessibility
